Live Data Feed with KML

The above map was created using Google Maps API. The purpose was to link a live KML (Keyhole Markup Language) to the map. The KML I chose was weekly earthquakes with a magnitude of 1.0 or greater from the USGS. Some advantages to using a KML is that the data is already symbolized and interactive. The symbology is also a disadvantage because it cannot be altered or customized.

Live Data Feed with GeoJSON

The above map was created using Google Maps API. The purpose was to link a live GeoJSON, another data format, to the map. The GeoJSON I chose was all the earthqaukes that have occured in the past week again taken from the USGS website. An advantage to using a GeoJSON is that the symbology is completely customizable. A disadvantage is that it can take a lot of code to customize the data which could generate multiple errors and become confusing.

Live Data Feed using Esri API

The above map was created using Esri's API. This time I used another live KML to link to the map. Esri's API is a little simpler to use and has widgets and functions built in to help with customization. Again, the KML was already symbolized and not able to be altered in this API however, the original symbology was rendered in a more clarifying manner.
